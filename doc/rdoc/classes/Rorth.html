exec(code)

Interprets rorth code. Expects an array. Needs some cleaning up. 

     code = "2 2 +".split                # => ['2', '2', '+']
     exec code # => Depends on input

An interesting feature/bug is that once a piece of code has been appraised for one thing, something that doesn‘t jump around (ifs, words, and loops,) you could theoretorically have a piece of code do 2 things. For instance, if you had a word called ""word"" (Yes, that is "word" surrounded by quotations marks) you could push "word" to the stack and execute a command called "word"
